最小 Coding Agent
是怎么 跑起来 的

mini-pi 是一个学习用的终端 coding agent： 极简 system prompt + 四个工具 + 一个 tool-calling 循环。 灵感来自 Pi， 非官方、非附属。

看源码 看 Agent Loop →

本页解释仓库里的真实代码结构。Agent 能执行 shell、改文件——只在可信工作区使用。

01 为什么做这么「少」

市面上的 coding agent 功能很多：plan mode、subagent、MCP、权限弹窗、花哨 TUI…… Pi 的有趣之处在于反其道而行——内核刻意做小，复杂能力用扩展补。

mini-pi 更进一步：只保留「能改代码」的最小闭环，方便你看清 所有 agent harness 共享的骨架：

Model + Prompt + Tools + Loop = Coding Agent
没有魔法。模型决定下一步；本地代码执行工具；结果塞回对话；重复直到模型闭嘴。
  • 短 prompt —— 少占上下文，少绑死行为
  • 四个工具 —— 读写改文件 + 跑命令，够改代码了
  • 信任模型 —— 不堆业务规则，把流程交给 LLM

02 源码地图

整个项目几乎只有这些文件（外加配置）：

src/
├── index.ts      # CLI：REPL / 单次 -p
├── agent.ts      # ★ Agent loop（核心）
├── tools.ts      # read / write / edit / bash
├── llm.ts        # OpenAI 兼容客户端（默认 xAI）
├── system.ts     # 极简 system prompt
└── tools.test.ts # 工具冒烟测试（无 API）
入口

index.ts

解析参数、读 API key、交互输入，把用户消息交给 runAgent。

大脑

agent.ts

维护 messages[]，反复调模型；有 tool_calls 就执行再塞回。

手脚

tools.ts

工具的 JSON Schema（给模型看）+ 本地执行实现（真正改磁盘）。

通道

llm.ts

用 OpenAI SDK 连 api.x.ai 或任意兼容端点。

03 Agent Loop（全部核心）

下面是 runAgent 的逻辑。点「下一步」可走一遍一次典型任务： 「创建一个 hello.ts」。

1

组装 messages

system prompt +（可选）历史 + 本轮 user 消息。这是模型能看到的全部上下文。

2

调用模型 chat.completions.create

带上 tools 定义，tool_choice: "auto"——让模型自己决定要不要调用工具。

3

收到 assistant 回复

可能有文字，也可能有 tool_calls（例如 write hello.ts）。把 assistant 消息 append 进数组。

4

有 tool_calls？→ 本地执行

executeTool(cwd, name, args) 真正写文件 / 跑 bash。结果以 role: "tool" 写回 messages。

5

回到步骤 2

模型带着工具结果再推理。可能继续 read/edit/bash，也可能只输出总结文字。

✓

没有 tool_calls → 结束

返回更新后的 history。CLI 把最后的文字打印给你。默认最多 30 轮防死循环。

一次真实的 messages 长什么样

简化示意（不是完整 JSON）：

system
You are a coding agent… four tools: read, write, edit, bash…
user
Create hello.ts that prints hello
assistant · tool_call
write({ path: "hello.ts", content: "console.log('hello')" })
tool · write
Wrote 24 chars to /…/hello.ts
assistant
Created hello.ts that prints "hello".

关键：模型从不直接碰磁盘。它只输出结构化请求； 你的 tools.ts 才是真正改世界的那一层。

04 四个工具

每个工具有两面：Schema（描述给模型）和 实现（在 Node 里执行）。 模型看到的是名字、说明和 JSON 参数；它填参数，你跑代码。

fs.readFile + 行号

read

读文件，可带 offset/limit。返回带行号的内容，方便模型引用。

fs.writeFile

write

创建或覆盖整文件，自动建父目录。适合新文件或大改。

精确字符串替换

edit

old_string → new_string。默认只允许匹配一次（防误伤）。

bash -lc

bash

在工作区跑 shell：ls、git、测试……有超时与输出截断。

为什么是「精确替换」而不是 diff

和 Pi / Claude Code 一类工具一样：让模型给出能在文件里唯一命中的原文再替换。 简单、可预测；匹配 0 次或多次会报错，逼模型 read 后再试。

05 关键代码片段

System prompt（几乎就这么短）

export function buildSystemPrompt(cwd: string): string {
  return `You are a coding agent in a terminal. Work in: ${cwd}

You have four tools: read, write, edit, bash.
- Prefer edit over rewrite for existing files.
- Use bash for shell commands (ls, git, tests, etc.).
- Be concise. Solve the task; don't narrate unless asked.
- After tools finish, give a short summary of what you did.`;
}

Loop 的心脏

for (let turn = 0; turn < maxTurns; turn++) {
  const response = await client.chat.completions.create({
    model,
    messages,
    tools: toolDefinitions,
    tool_choice: "auto",
  });

  const msg = response.choices[0].message;
  messages.push({ role: "assistant", content: msg.content, tool_calls: msg.tool_calls });

  const toolCalls = msg.tool_calls ?? [];
  if (toolCalls.length === 0) return messages;  // 模型做完了

  for (const call of toolCalls) {
    const result = await executeTool(cwd, call.function.name, call.function.arguments);
    messages.push({ role: "tool", tool_call_id: call.id, content: result.output });
  }
  // 带着 tool 结果再进下一轮 for
}

06 刻意没有的东西

不是做不到，是学习版先砍掉，避免核心被淹没：

Session 落盘 / resume
Token streaming
权限沙箱 / 审批弹窗
MCP / Skills / Extensions
Subagent / Plan mode
富文本 TUI
上下文自动压缩
多模型热切换 UI

真正的 Pi 用扩展系统把这些能力加回去；OpenClaw 等产品则在类似内核上接聊天通道。
