How a minimal coding agent
actually runs

mini-pi is a learning-oriented terminal coding agent: a minimal system prompt + four tools + one tool-calling loop. Inspired by Pi, unofficial and unaffiliated.

View source See the agent loop →

This page explains the real code structure in the repo. The agent can run shell commands and edit files — use it only in trusted workspaces.

01 Why so little

Most coding agents ship a lot: plan mode, subagents, MCP, permission prompts, fancy TUIs… Pi is interesting because it goes the other way — the core stays small on purpose, and complexity lives in extensions.

mini-pi goes further: it keeps only the minimal closed loop that can change code, so you can see clearly the skeleton every agent harness shares:

Model + Prompt + Tools + Loop = Coding Agent
No magic. The model chooses the next step; local code runs tools; results go back into the conversation; repeat until the model stops.
  • Short prompt — less context burned, fewer hard-coded behaviors
  • Four tools — read, write, edit files + run commands; enough to change code
  • Trust the model — no pile of business rules; leave the flow to the LLM

02 Source map

The whole project is basically these files (plus config):

src/
├── index.ts      # CLI: REPL / one-shot -p
├── agent.ts      # ★ Agent loop (core)
├── tools.ts      # read / write / edit / bash
├── llm.ts        # OpenAI-compatible client (default xAI)
├── system.ts     # minimal system prompt
└── tools.test.ts # tool smoke tests (no API)
Entry

index.ts

Parses args, reads the API key, takes interactive input, and hands user messages to runAgent.

Brain

agent.ts

Owns messages[], calls the model repeatedly; when there are tool_calls, executes them and appends results.

Hands

tools.ts

Tool JSON Schema (what the model sees) + local execution (what actually touches disk).

Channel

llm.ts

Uses the OpenAI SDK against api.x.ai or any compatible endpoint.

Simplified sketch (not full JSON):

Key point: the model never touches the disk directly. It only emits structured requests; your tools.ts is the layer that actually changes the world.

04 The four tools

Every tool has two faces: a schema (described to the model) and an implementation (run in Node). The model sees names, descriptions, and JSON parameters; it fills them in, and you run the code.

fs.readFile + line numbers

read

Read a file, optional offset/limit. Returns numbered lines so the model can cite them.

fs.writeFile

write

Create or overwrite a whole file; creates parent dirs. Good for new files or big rewrites.

exact string replace

edit

old_string → new_string. By default only one match is allowed (avoids collateral damage).

bash -lc

bash

Run shell in the workspace: ls, git, tests… with timeout and output truncation.

Why exact replace instead of a diff

Same idea as tools like Pi / Claude Code: the model provides original text that uniquely matches in the file, then we replace it. Simple and predictable; 0 or many matches error out, which forces the model to read and try again.

05 Key code snippets

System prompt (almost this short)

export function buildSystemPrompt(cwd: string): string {
  return `You are a coding agent in a terminal. Work in: ${cwd}

You have four tools: read, write, edit, bash.
- Prefer edit over rewrite for existing files.
- Use bash for shell commands (ls, git, tests, etc.).
- Be concise. Solve the task; don't narrate unless asked.
- After tools finish, give a short summary of what you did.`;
}

The heart of the loop

for (let turn = 0; turn < maxTurns; turn++) {
  const response = await client.chat.completions.create({
    model,
    messages,
    tools: toolDefinitions,
    tool_choice: "auto",
  });

  const msg = response.choices[0].message;
  messages.push({ role: "assistant", content: msg.content, tool_calls: msg.tool_calls });

  const toolCalls = msg.tool_calls ?? [];
  if (toolCalls.length === 0) return messages;  // model is done

  for (const call of toolCalls) {
    const result = await executeTool(cwd, call.function.name, call.function.arguments);
    messages.push({ role: "tool", tool_call_id: call.id, content: result.output });
  }
  // next for-iteration with tool results in context
}

06 Intentionally missing

Not because we can’t — the learning cut keeps the core uncluttered:

Session persistence / resume
Session tree / branch
Permission sandbox / approval UI
MCP / Skills / Extensions
Subagent / Plan mode
Rich TUI
Exact tokenizer billing
Multi-model hot-switch UI

Real Pi layers these back on via an extension system; products like OpenClaw attach chat channels on a similar core.
