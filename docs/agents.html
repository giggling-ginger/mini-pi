项目规则怎么进
模型的 system prompt

mini-pi 的 system prompt 故意很短。项目约定（包管理器、测试命令、目录结构……） 不写死在代码里，而是读仓库里的 AGENTS.md，拼到 system 后面。

源码：src/context-files.ts · src/system.ts

01 为什么需要它

没有 AGENTS.md

模型只知道「你是 coding agent + 四个工具」。 不知道你们用 pnpm 还是 npm，测试怎么跑，哪些目录别乱动。

有 AGENTS.md

启动时读进 system prompt。每一轮 API 调用都带着这些规则 （system 不进 session 文件，但每次会重新加载最新磁盘内容）。

和 Session 的分工： Session 记「我们聊过什么」；AGENTS.md 记「这个仓库的规矩」。 前者是对话历史，后者是项目说明书。

02 怎么「发现」文件

从 cwd 开始，一路走到 git 根目录（有 .git 的那层），沿途找：

  • AGENTS.md（优先）
  • 否则 CLAUDE.md
  • 否则 AGENT.md

每个目录只取一个；更靠近 cwd 的文件排在后面（更具体）。

repo/                          # git root = 停止向上
├── AGENTS.md                  # ① 先收集（全局约定）
├── package.json
└── packages/app/              # 你在这里跑 mini-pi
    ├── AGENTS.md              # ② 后收集（包级约定）
    └── src/...
预算：全部 context 文件合计最多约 32KB 字符，超出截断并标注 [truncated]，避免单个巨型 markdown 撑爆上下文。

03 怎么塞进 prompt

每一轮发给模型的消息大致是这样一块「三明治」：

system · base（代码写死，很短）
You are a coding agent… four tools: read, write, edit, bash…
system · project instructions（AGENTS.md 拼上）
# Project instructions ### AGENTS.md Use pnpm. Run tests with vitest… ### packages/app/AGENTS.md Prefer colocated tests…
history（来自 session，不含 system）
user / assistant / tool / …
本轮 user
add a failing test for parseArgs
// system.ts（简化）
const files = loadContextFiles({ cwd })
const prompt =
  buildBaseSystemPrompt(cwd) +
  formatContextBlock(files)   // 没有文件则是空字符串

messages = [
  { role: "system", content: prompt },
  ...history,
  { role: "user", content: userMessage },
]

因此：你改了 AGENTS.md 再开一轮（或新 session），模型立刻读到新规则—— 不必清 session（system 每次重建）。

05 AGENTS.md 写什么好

适合写

包管理器、测试/构建命令、目录约定、禁止事项、PR/commit 风格、和本仓库相关的工具偏好。

别塞爆

整本设计文档、巨型 API 列表、会过期的临时 TODO——会占上下文且干扰模型。保持短、可执行。

06 CLI / REPL

命令作用
（默认） 自动加载 AGENTS.md 等，启动时打印路径
--no-context-files / -nc 本 run 不加载（对照实验很有用）
REPL /context 列出当前加载了哪些文件和字符数 
npm run dev -- -p "跑一下 typecheck"
# stderr 可能看到: context 1 file(s): AGENTS.md

npm run dev -- -nc -p "同一任务但不读 AGENTS.md"

07 常见误解

误解实际
AGENTS.md 会存进 session JSONL 不存。system 每次现拼，改文件立即生效
和 skills / MCP 一样 只是文本说明书，不注册新工具
模型一定会 100% 遵守 仍是 soft instructions；写清楚 + 简短更有效
子目录 CLAUDE.md 和 AGENTS.md 会合并 同目录只取优先级最高的一个名字
一句话：AGENTS.md = 可版本管理的项目 system 附录。 Harness 负责找到它并拼进 prompt；写得好不好，决定 agent 像不像「你们队的人」。
